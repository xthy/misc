## 1. Cold Call Script (Outbound SDR)

### 오프닝 (첫 15초가 결정)

**패턴 인터럽트 방식** (전형적 세일즈 콜과 다르게 시작):

```
"[Name]님, 안녕하세요. [Your Name]입니다, [Company]에서 전화드렸습니다.
솔직히 말씀드리면 콜드콜이에요.
30초만 주시면, 관련 없으시면 바로 끊겠습니다. 괜찮으시겠어요?"
```

> **원칙**: 정직함이 호기심을 유발합니다. "잠깐 시간 되세요?"보다 전환율 2-3x 높음.

### 가치 가설 (30초)

```
"[Company Name]이 최근 [trigger: 채용 확대/자금 조달/제품 출시]하신 것 봤습니다.
보통 이 단계에서 [industry] 기업들이 겪는 문제가 [specific problem]인데요.
혹시 [their company]에서도 이 부분이 과제로 느껴지시는지 궁금해서요."
```

### 반응별 대응

| 반응 | 대응 |
|------|------|
| "네, 맞아요" | "구체적으로 어떤 부분이 가장 큰 과제인가요?" → Discovery로 전환 |
| "좀 다른데요" | "어떤 부분이 다르신가요? 오히려 더 듣고 싶습니다" → 수정된 가설로 진행 |
| "지금 바쁩니다" | "이해합니다. 15분 미팅을 잡으면 될까요? [2-3개 시간 옵션]은 어떠세요?" |
| "관심 없습니다" | "알겠습니다. 혹시 [problem area]를 담당하시는 다른 분을 소개해주실 수 있나요?" |
| "자료 보내주세요" | "어떤 내용이 가장 도움 되실까요? 한 가지만 여쭤봐도 될까요 — [qualifying question]" |

### 미팅 잡기

```
"[Problem]에 대해 [peer company]가 어떻게 해결했는지 15분 정도 공유드리면
판단하시는 데 도움이 될 것 같은데, 이번 주 [화요일 / 목요일] 오전은 어떠세요?"
```

### 보이스메일 (30초 이내)

```
"[Name]님, [Company]의 [Your Name]입니다.
[Trigger event]에 대해 간단히 말씀드리고 싶어 전화드렸습니다.
[Email / LinkedIn]으로도 연락드리겠습니다.
[전화번호], 다시 한번, [전화번호]입니다."
```

## 2. Discovery Call Framework (AE)

### 미팅 세팅 (첫 5분)

```
"오늘 미팅 잡아주셔서 감사합니다.
오늘 제가 이해하고 싶은 건:
1. 현재 [area] 관련해서 어떤 상황이신지
2. 어떤 부분을 개선하고 싶으신지
3. 저희가 도움 될 수 있는 부분이 있는지

저도 [Company] 소개를 간략히 드리겠지만,
먼저 [Name]님 상황을 충분히 듣고 싶습니다.

그 전에 — 오늘 이 미팅에서 가장 얻고 싶은 것이 무엇인가요?"
```

### SPIN + Gap Selling 통합 질문 플로우

#### Phase 1: Situation (현재 환경) — 5-7분

| # | 질문 | 목적 |
|---|------|------|
| S1 | "현재 [area]를 어떤 방식으로 처리하고 계신가요?" | 현재 프로세스 이해 |
| S2 | "이 업무에 어떤 도구/시스템을 사용하시나요?" | 기술 환경 파악 |
| S3 | "팀 구성은 어떻게 되나요? 몇 명이 관여하시나요?" | 규모/복잡도 파악 |
| S4 | "현재 이 분야의 KPI나 목표 수치가 있으시다면요?" | Baseline 확보 |
| S5 | "이 프로세스가 마지막으로 크게 바뀐 건 언제인가요?" | 변화 수용도 판단 |

> **주의**: Situation 질문은 5개 이내로. 많으면 인내심이 떨어짐. 사전 리서치로 대체 가능한 건 미리 파악.

#### Phase 2: Problem (문제 식별) — 7-10분

| # | 질문 | 목적 |
|---|------|------|
| P1 | "현재 방식에서 가장 불편하거나 비효율적인 부분은요?" | 표면적 Pain |
| P2 | "이상적이지 않다고 느끼시는 구체적 상황을 하나 말씀해주시겠어요?" | 구체적 사례 확보 |
| P3 | "팀에서 가장 많은 불만을 표현하는 부분은 어디인가요?" | 팀 차원 Pain |
| P4 | "이 문제가 얼마나 자주 발생하나요?" | 빈도/심각도 |
| P5 | "수동으로 해결하고 계신 workaround가 있나요?" | 숨겨진 비용 |
| P6 | "이전에 이 문제를 해결하려고 시도하신 적이 있나요? 결과는요?" | 과거 시도 학습 |

#### Phase 3: Implication (영향 확대) — 10-12분 ⭐ 가장 중요

| # | 질문 | 목적 |
|---|------|------|
| I1 | "그 문제가 매출/파이프라인에 직접적으로 어떤 영향을 미치나요?" | 재무적 임팩트 |
| I2 | "팀이 [workaround]에 쓰는 시간을 주당으로 환산하면요?" | 시간 비용 정량화 |
| I3 | "이 문제가 해결 안 되면 6-12개월 후에는 어떻게 될까요?" | 미래 리스크 |
| I4 | "다른 부서나 팀에도 영향이 가나요? 어떤 식으로요?" | 영향 범위 확대 |
| I5 | "경영진이 이 문제를 인지하고 있나요? 어떤 반응이셨나요?" | EB awareness |
| I6 | "경쟁사 대비 이 영역에서 뒤처지고 있다고 느끼시나요?" | 경쟁 압박 |
| I7 | "올해 목표를 달성하는 데 이 문제가 걸림돌이 되나요?" | 전략적 중요도 |

> **코칭 포인트**: 이 단계에서 침묵을 두려워하지 말 것. 질문 후 3초 이상 기다리면 더 깊은 답변이 나옴.

#### Phase 4: Need-Payoff (해결 가치) — 5-7분

| # | 질문 | 목적 |
|---|------|------|
| N1 | "만약 이 문제가 완전히 해결된다면, 팀의 하루가 어떻게 달라질까요?" | 비전 그리기 |
| N2 | "이 시간이 절약된다면 어디에 재투자하시겠어요?" | 가치 확인 |
| N3 | "[specific improvement]가 가능하다면, 올해 목표에 어떤 영향이 있을까요?" | 목표 연결 |
| N4 | "이 성과를 경영진에게 보고하신다면, 어떤 반응이실 것 같나요?" | EB 가치 확인 |
| N5 | "이 문제를 해결하는 것이 [Name]님 개인적으로도 중요한가요?" | Champion 동기 |

### 미팅 마무리 (마지막 5분)

```
"오늘 말씀 정리하면:
- 현재 [current state 요약]
- 핵심 과제는 [pain 1, 2]
- 이를 해결하면 [expected impact]

맞으시죠?

다음 단계로 저희가 [demo / 기술 검토 / 제안서]를 준비해드리면 좋겠는데,
그 전에 몇 가지만 더 여쭤봐도 될까요:

1. 이런 솔루션에 대한 예산이 이미 잡혀 있나요, 아니면 확보가 필요한가요?
2. 최종 결정은 누가 하시나요?
3. 언제까지 결정하셔야 하는 타임라인이 있으신가요?
4. 다음 미팅에 함께 참석하면 좋을 분이 있으신가요?

좋습니다. 그러면 [다음 단계]를 [날짜]까지 준비해서 공유드리겠습니다.
[Name]님 쪽에서는 [action item]을 확인해주시면 좋겠습니다."
```

## 3. Demo Script Framework (AE + SE)

### Pre-Demo Checklist
- [ ] Discovery에서 파악한 Pain 1-3개 정리
- [ ] 고객의 용어/표현으로 시나리오 구성 (우리 용어가 아닌)
- [ ] "So what?" 테스트: 각 기능이 고객 Pain에 어떻게 연결되는지 한 문장으로 설명 가능한가?
- [ ] 참석자 확인 — 새 참석자 있으면 Discovery 질문 추가
- [ ] Demo 환경 사전 테스트

### Demo 구조

```
1. [3분] 컨텍스트 재확인
   "지난번 대화에서 [pain 1, 2, 3]이 핵심이셨는데,
    오늘은 이 부분에 집중해서 보여드리겠습니다.
    그 후에 말씀하신 것 외에 추가로 보고 싶은 것이 있으시면
    알려주세요."

2. [20분] Pain 중심 시연
   Pain 1 → Feature A가 어떻게 해결하는지 (before/after)
   Pain 2 → Feature B 시연 (고객 시나리오 기반)
   Pain 3 → Feature C 시연

   ★ 각 Pain마다:
   "이게 [their current workaround]를 대체하는 건데,
    예상하시는 것처럼 동작하나요?
    팀에서 이걸 사용한다면 어떤 변화가 있을까요?"

3. [10분] 질문 & 반응
   "지금까지 본 것 중 가장 관심 가는 부분은 어디였나요?"
   "실제로 적용한다면 어떤 장애물이 예상되시나요?"
   "다른 팀/부서에서도 활용할 만한 부분이 있을까요?"

4. [5분] Next Steps
   "다음으로는 [PoC / 제안서 / 기술 검토]가 좋을 것 같은데,
    [Name]님 생각은 어떠세요?"
   "[EB name]도 이 내용을 보시면 좋을 것 같은데,
    별도로 간략한 Executive briefing을 드릴까요?"
```

## 4. QBR Call Framework (CS + AE)

### QBR Agenda Script

```
"오늘 QBR에서는 세 가지를 다루겠습니다:

1. 지난 분기 성과 리뷰와 합의한 KPI 달성 현황
2. 앞으로 더 가치를 높일 수 있는 방법
3. 다음 분기 함께 달성할 목표

먼저, [Name]님 팀 입장에서 지난 분기 [our product] 활용에 대한
전반적인 소감부터 들어볼까요?"
```

### QBR 핵심 질문

| 섹션 | 질문 |
|------|------|
| **성과** | "합의했던 [KPI]에서 어느 정도 달성 되셨나요?" |
| **가치** | "[Our product] 없이 이 업무를 한다면 어떤 차이가 있을까요?" |
| **불만** | "개선되었으면 하는 부분이 있다면요?" |
| **Adoption** | "[Feature X]는 아직 많이 활용하지 않으시는 것 같은데, 이유가 있으시나요?" |
| **Expansion** | "다음 분기에 [new capability]를 시도해볼 의향이 있으신가요?" |
| **전략** | "내년 팀의 핵심 우선순위 3가지가 있다면요?" |
| **관계** | "저희 지원에 대해 만족하고 계신가요? 더 필요한 게 있으시다면요?" |

## 5. Objection Handling 빠른 참조표

| 반론 | 카테고리 | 즉석 대응 | 후속 질문 |
|------|---------|----------|----------|
| "너무 비쌉니다" | Price | "가격과 가치 중 어디에 걱정이신 건지 여쭤봐도 될까요?" | "현재 이 문제의 비용을 금액으로 환산하면 얼마인가요?" |
| "이미 다른 걸 씁니다" | Competition | "이해합니다. [경쟁사]로 [specific area]는 어떻게 해결하고 계신가요?" | "[known weakness]에 대해서는 만족하고 계신가요?" |
| "생각해 보겠습니다" | Stall | "물론이죠. 혹시 내부에서 추가로 확인하셔야 하는 점이 있으신가요?" | "이번 주 내로 확인 후 연락 주시면, 그때 한번 더 통화할까요?" |
| "상사 확인이 필요해요" | Authority | "그 분이 가장 궁금해하실 포인트는 무엇일까요?" | "제가 Executive Summary를 준비해드릴까요?" |
| "전에 비슷한 거 실패했어요" | Past failure | "어떤 부분이 안 됐었나요? 기술? 도입 과정? 사용 측면?" | "그때와 지금 상황이 다른 점은 무엇인가요?" |
| "내부 개발하겠습니다" | Build vs Buy | "내부 개발의 총 비용(인력, 시간, 유지보수)을 산출해보셨나요?" | "코어 제품 개발에 집중하는 대신 이걸 아웃소싱하면?" |
| "기능 X가 없네요" | Feature gap | "[Feature X]로 달성하려는 구체적인 결과가 무엇인가요?" | "그 기능 없이 현재 고객들이 어떻게 해결하는지 보여드릴까요?" |
| "도입이 복잡할 것 같아요" | Implementation | "보통 [X주] 안에 라이브하고 있습니다. 저희가 [Y%]를 대신합니다." | "6개월 뒤에도 현재 방식으로 하시는 것과 비교하면요?" |
| "자료 보내주세요" | Brush-off | "어떤 내용이 가장 도움 되실까요?" | "[Problem]이 현재 실제로 해결해야 할 과제인가요?" |
| "계약이 남았어요" | Locked-in | "갱신 시기가 언제이신가요?" | "3-6개월 전에 비교 평가하시면 협상력이 올라가실 텐데요" |

## Agent 활용: Call Script Automation

### Pre-Call Briefing (Agent 자동 생성)

```
각 콜 전에 Agent가 자동으로 생성:

1. Account snapshot (산업, 규모, Tier, Health Score)
2. Contact profile (직급, 관심사, 최근 LinkedIn 활동)
3. Engagement history (이전 터치 이력, 마지막 대화 요약)
4. Recommended talking points (trigger event, peer reference)
5. MEDDICC gap → 이번 콜에서 채워야 할 정보
6. Suggested questions (gap 기반 2-3개)
```

### Post-Call Processing (Agent 자동 실행)

```
콜 완료 후 Agent가 자동으로:

1. 트랜스크립트에서 MEDDICC 요소별 정보 추출
2. CRM Activity 기록 (요약, 다음 스텝, 감정 분석)
3. MEDDICC 필드 업데이트 초안 생성
4. Next best action 추천
5. Follow-up 이메일 초안 생성 (콜 내용 기반)

→ AE가 확인 + 승인하면 CRM 반영
```
