# Objection Handling Guide

---

## 원칙

1. **반박하지 않는다.** 먼저 인정하고, 질문으로 진짜 이유를 파악한다.
2. **질문이 답보다 먼저.** 처방 전에 진단한다.
3. **침묵은 무기다.** 대응 후 멈추고, 상대방이 소화할 시간을 준다.
4. **모든 반론을 극복할 필요는 없다.** 일부는 진짜 disqualifier다.
5. **고객 이야기가 논리보다 강하다.** "저희 생각에는"보다 "[고객 A]의 경우"가 효과적이다.

---

## Objection Category Map

```
반론
├── 가격/예산 (Price/Budget)
│   ├── "너무 비쌉니다"
│   └── "예산이 없습니다"
├── 경쟁/대안 (Competition)
│   ├── "이미 [경쟁사] 쓰고 있습니다"
│   ├── "내부 개발하겠습니다"
│   └── "계약이 남아있습니다"
├── 타이밍/우선순위 (Timing)
│   ├── "지금은 아닙니다"
│   ├── "생각해 보겠습니다"
│   └── "다음 분기에 다시요"
├── 권한/프로세스 (Authority)
│   ├── "상사 확인이 필요합니다"
│   └── "여러 부서가 관여합니다"
├── 제품/기능 (Product)
│   ├── "기능 X가 없네요"
│   ├── "비슷한 거 실패한 적 있어요"
│   └── "도입이 복잡할 것 같아요"
└── 회피 (Brush-off)
    ├── "자료 보내주세요"
    └── "관심 없습니다"
```

## 상세 Objection Handling

### 1. "너무 비쌉니다" / "예산이 없습니다"

**진단 질문**:
- "가격 자체가 부담이신 건가요, 아니면 투자 대비 가치가 불확실하신 건가요?"
- "비교 기준이 있으신가요? (경쟁사 가격, 내부 기대치 등)"

**대응 프레임: Isolate → Quantify → Reframe**

| 상황 | 대응 |
|------|------|
| 가치 불확실 | "말씀하신 [problem]이 월 [X]시간, 연 [$Y] 비용이라고 하셨는데, 저희 투자금은 그 [fraction]입니다. 아무것도 안 하시는 비용이 더 큰 셈이죠." |
| 예산 미확보 | "ROI가 명확하면 예산을 확보할 수 있는 프로세스가 있으신가요? 저희가 비즈니스 케이스를 함께 만들어드릴 수 있습니다." |
| 경쟁사 대비 비쌈 | "가격 외에 평가하시는 기준 중 저희가 강한 부분은 어디라고 보시나요? 총소유비용(TCO)으로 비교해보시면 다른 결론이 나올 수 있습니다." |

**절대 하지 말 것**: 즉시 할인 제안. (가격 저항 ≠ 할인 요청)

---

### 2. "이미 [경쟁사]를 쓰고 있습니다"

**진단 질문**:
- "[경쟁사]로 [specific area]는 어떻게 해결하고 계신가요?"
- "100% 만족하시나요, 아니면 아쉬운 부분이 있으신가요?"

**대응**:

```
"이해합니다. [경쟁사]는 [acknowledged strength]에 강한 솔루션이죠.

저희 고객 중 [경쟁사]에서 전환하신 분들이 말씀하시는 건,
[specific gap: 확장성 / 통합 / 사용성 / 분석]에서 차이를 느꼈다는 거예요.

혹시 [known weakness area]는 현재 어떻게 처리하고 계신가요?"
```

**핵심**: 경쟁사를 비난하지 않는다. 차이점을 질문으로 탐색한다.

---

### 3. "생각해 보겠습니다" / "지금은 아닙니다"

**진단 질문**:
- "완전히 이해합니다. 한 가지만 여쭤봐도 될까요 — 타이밍 문제인가요, 아니면 제가 충분히 설명 못 드린 부분이 있는 건가요?"
- "내부에서 추가로 확인하셔야 하는 구체적인 사항이 있으신가요?"

**대응 분기**:

| 진짜 이유 | 대응 |
|----------|------|
| 타이밍 문제 (진짜) | "언제쯤 다시 논의하면 좋을까요? 그때 [relevant resource]를 준비해두겠습니다." |
| 내부 설득 필요 | "어떤 자료가 있으면 내부 논의에 도움 되실까요? Executive summary를 만들어드릴까요?" |
| 숨겨진 반론 | "솔직히 여쭤볼게요 — 진행을 망설이시게 하는 가장 큰 이유가 무엇인가요?" |
| 거절 (정중한) | "알겠습니다. [3개월 뒤] 즈음에 한번 체크인 드려도 괜찮을까요?" |

---

### 4. "상사 확인이 필요합니다"

**진단 질문**:
- "그 분이 가장 궁금해하실 포인트는 무엇일까요?"
- "비슷한 결정을 하실 때 보통 어떤 기준으로 판단하시나요?"

**Champion Coaching 대응**:

```
"물론이죠. 제가 도움드릴 수 있는 부분이 있습니다:

1. Executive Summary 1장으로 핵심을 정리해드릴까요?
   → [First Name]님이 그대로 공유하시면 됩니다.

2. 혹시 그 분과 직접 짧은 미팅을 잡을 수 있을까요?
   기술적 질문에 바로 답변드리면 결정이 빨라질 수 있습니다.

3. [First Name]님이 내부에서 설명하실 때,
   가장 강조하면 좋을 포인트가 [ROI 수치, peer reference]입니다."
```

---

### 5. "전에 비슷한 거 실패했어요"

**진단 질문**:
- "어떤 부분이 안 됐었나요? 기술? 도입 과정? 사용자 채택?"
- "그때 경험에서 '이건 달랐으면 좋겠다'고 느끼신 점이 있나요?"

**대응**:

```
"그 경험이 중요한 맥락이 되네요. 감사합니다.

실패 원인이 [their answer: 도입 과정]이었다면,
저희는 그 부분을 이렇게 다르게 접근합니다:

• [Specific differentiator 1]
• [Specific differentiator 2]

[Similar Company]도 이전에 [competitor/solution] 도입에 실패한 후
저희와 다시 시도해서 [specific result]를 달성했습니다.

차이점을 직접 확인하실 수 있도록
[PoC / 파일럿 / 30분 deep dive]를 제안드려도 될까요?"
```

---

### 6. "내부 개발하겠습니다"

**진단 질문**:
- "내부 개발의 타임라인과 투입 인력은 어떻게 잡고 계신가요?"
- "유지보수와 업데이트까지 포함한 총 비용을 산출해보셨나요?"

**TCO 프레임 대응**:

```
"내부 개발은 확실히 완전한 통제권을 갖는 장점이 있죠.

보통 고객사에서 비교하실 때 간과되는 부분이 있는데:

1. 개발 시간: 보통 6-12개월 vs 저희는 [X주] 내 라이브
2. 유지보수: 전담 인력 [N명] × 연봉 vs 저희 구독료
3. 기회비용: 엔지니어링 팀이 코어 제품 대신 이걸 만드는 비용

[Your Company]의 핵심 경쟁력이 [their core product] 아닌가요?
이걸 직접 만드는 것이 그 핵심에 집중하는 것과 양립 가능한가요?"
```

---

### 7. "기능 X가 없네요"

**진단 질문**:
- "그 기능으로 달성하려는 구체적인 워크플로우나 결과가 무엇인가요?"
- "그 기능이 없으면 다른 모든 가치에도 불구하고 진행이 어려운 건가요?"

**대응 분기**:

| 상황 | 대응 |
|------|------|
| 로드맵에 있음 | "[Feature X]는 [timeframe]에 출시 예정입니다. 현재 고객들은 [workaround]로 해결하고 있고, [result]를 내고 있습니다." |
| 로드맵에 없음 (nice-to-have) | "솔직히 그 기능은 현재 없습니다. 다만 [Name]님이 말씀하신 [underlying need]는 [alternative approach]로 달성 가능합니다." |
| 로드맵에 없음 (dealbreaker) | "현재 그 부분은 저희 역량 밖입니다. 솔직히 말씀드리는 것이 맞다고 생각합니다. 다만 [alternative]로 [partial solution]은 가능한데, 그것만으로 충분하지 않으시다면 이해합니다." |

**핵심**: 없는 기능이 있다고 거짓말하지 않는다. 정직함이 신뢰를 만든다.

---

### 8. "도입이 복잡할 것 같아요"

**대응**:

```
"합리적인 우려입니다.

실제 도입 과정은:
• 기간: 평균 [X주], 최대 [Y주]
• 저희가 처리하는 부분: [데이터 마이그레이션, 설정, 통합]의 [Z%]
• 고객 측 필요 시간: 주당 [N시간] × [M주]
• 전담 지원: CSM + 기술 지원 엔지니어 배정

[Similar Company]는 [X주] 만에 라이브했고,
[first value milestone]까지 [Y일] 걸렸습니다.

도입 계획서를 미리 공유드릴 수 있는데,
보시면 구체적인 범위가 느껴지실 거예요."
```

---

### 9. "자료 보내주세요" (Brush-off)

**진단**:

```
"물론이죠. 적절한 자료를 보내드리려면 한 가지만 여쭤봐도 될까요 —

[Problem area]가 현재 실제로 해결해야 할 과제인가요,
아니면 나중에 참고로 보시려는 건가요?"
```

| 반응 | 대응 |
|------|------|
| "실제 과제입니다" | "그러면 어떤 내용이 가장 도움 되실까요? [Option A] vs [Option B]?" → 자료 발송 + 리뷰 미팅 제안 |
| "참고로요" | "알겠습니다. [brief resource]를 보내드리겠습니다. 타이밍 되실 때 알려주세요." → Nurture로 이동 |

---

### 10. "계약이 남아있어요"

**대응**:

```
"갱신 시기가 언제이신가요?"

[3개월 이내]: "그러면 지금이 비교 평가하시기 좋은 시점이네요.
갱신 전에 대안을 확보하시면 협상력이 올라갑니다."

[6개월+]: "이해합니다. 보통 갱신 3-6개월 전에 평가를 시작하시는 게
좋은데, [time]쯤 다시 연락드려도 괜찮을까요?

한 가지 — 현재 계약에 포함 안 되는 [use case]가 있다면
병행해서 도입하시는 것도 가능합니다."
```

## Agent 활용: Real-time Objection Support

### 라이브 콜 중 Agent 지원 (향후 구현)

```
Agent가 실시간 콜 트랜스크립트를 분석하여:

1. 고객 발언에서 반론 유형 자동 분류
2. 해당 반론의 추천 대응을 화면에 표시
3. 관련 고객 사례/데이터 즉시 제공
4. 콜 후 반론 로그를 CRM에 자동 기록

→ Rep의 실시간 판단을 보조하되, 최종 대응은 항상 사람이 결정
```

### 반론 패턴 분석 (분기별)

```
Agent가 CRM Activity + Call transcript에서 반론 데이터를 집계:

1. 반론 유형별 빈도 (Top 5)
2. 반론 → 극복 → 진행 성공률 (유형별)
3. Rep별 반론 대응 성공률 비교
4. 새로운 반론 패턴 감지
5. Playbook 업데이트 제안

→ Win/Loss Analysis (Play 05)에 피드
```
